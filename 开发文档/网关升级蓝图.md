### 🗺️ 二、 优化后的升级路线图 (Roadmap 5.0)

接下来的重点非常清晰：**把前端页面上的“假按钮”变成“真功能”**，然后回过头来做 Redis 增强。

#### 🥇 第一优先级：前端控制台功能补全 (Dashboard Functionality)
*   **目标**：让截图里的“新增路由”、“编辑”、“删除”按钮能用。
*   **动作**：
    1.  **实现删除**：点击表格行的“删除”，调用后端 `DELETE /admin/routes/{id}`。
    2.  **实现新增**：点击“新增路由”，弹出一个表单模态框 (Dialog)，填写 ID、URI、断言规则，提交给后端 `POST` 接口。
    3.  **实现编辑**：回显数据并更新。

#### 🥈 第二优先级：流量驾驶舱 (Metrics Dashboard)
*   **目标**：在首页展示图表，而不是冷冰冰的表格。
*   **动作**：
    1.  引入 **ECharts**。
    2.  调用后端 `/actuator` 接口。
    3.  画出两个图：**JVM 内存水位图** (仪表盘) 和 **QPS 趋势图** (折线图)。

#### 🥉 第三优先级：Redis 基础设施回归 (Advanced Backend)
*   **目标**：引入 Redis 做防重放和缓存。
*   **动作**：
    1.  启用 Redis，封装工具类。
    2.  实现防重放拦截器。

### 🧠 今日复盘 (Digestion)

你在休息时，可以回想一下今天打通的这条“超级链路”：

1.  **浏览器** (Vue) 发起 `GET` 请求。
2.  **网关** (9000端口) 收到请求，**CORS 过滤器** 允许跨域。
3.  **RouteController** 接收请求，去 **Nacos** 拉取 JSON 配置。
4.  **Nacos** 返回配置 -> **Controller** 转成 List -> **前端** 拿到数据。
5.  **Vue** 拿到数据 -> **v-for** 渲染出你看到的表格。
